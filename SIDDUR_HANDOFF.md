## 1. ארכיטקטורה — איך הסידור עובד

זרימה: `app/tfilon/read.tsx` טוען leaves מ-`siddurTree` → מריץ `augmentLeavesForToday`
(הזרקות יום-ספציפיות) → מסנן ב-`isSectionRelevantToday` → טוען טקסט מ-Sefaria → מפרסר
ב-`parseParagraphs` → מרנדר.

**קבצים מרכזיים:**
- `app/tfilon/read.tsx` — הקומפוננטה הראשית. כאן: `isKedushahLeaf`, `isAmidahLeaf`,
  קולפס חזרת ש"ץ, `buildCuratedTOC` (כפתורי ניווט), טעינת leaves.
- `src/data/siddurAugment.ts` — הזרקות לכל יום (ר"ח, חנוכה, פורים, חוה"מ, צום...).
  `augmentLeavesForToday` = entry point. פר-נוסח handlers.
- `src/data/siddurRelevance.ts` — `isSectionRelevantToday`: מה מוצג מתי (תחנון, הלל,
  קריאת התורה, מוצ"ש...).
- `src/services/siddurParser.ts` — `parseParagraphs`: ממיר פסקאות Sefaria למבנה
  עם תגיות מותנות (בעשי"ת, בקיץ, יעלה ויבוא...). `markerToTags`, `shouldRender`,
  multi-paragraph conditional blocks (ברכת כהנים בתענית).
- `src/data/specialDayContent.ts` — טקסטים מסונתזים (ענינו, נחם, שושנת יעקב, ויבא
  עמלק, חנוכה נשא לפי יום). `CUSTOM_TEXTS` map.
- `src/data/specialDayLinks.ts` — באנרים (מגילה, הושענות, קינות, טלית ת"ב).
- `src/data/tefilaInserts.ts` — `getInsertsForDate`: רשימת התוספות בבאנר "תוספות לתפילה היום".
- `src/data/siddurIndex.json` — עץ הסידור המלא לכל הנוסחים (מקור: Sefaria).
- `src/hooks/useEffectiveDate.ts` — קורא `?fakeDate=YYYY-MM-DD` מה-URL לבדיקות.

**הרצת שרת לבדיקה:** PowerShell:
`Start-Process powershell -ArgumentList '-Command','cd ''C:\Users\ITSHA\yahadut-app''; npx expo start --web --port 8081' -WindowStyle Hidden`
(השרת קורס לפעמים — להפעיל מחדש. בודקים בדפדפן עם Chrome MCP.)

## 2. עובדה ארכיטקטונית קריטית — ההבדל בין אשכנז לספרד בספריא

- **`Siddur Ashkenaz`** → העמידה מחולקת ל-**22 תת-קטעים** (refs נפרדים: Patriarchs,
  Divine Might, Holiness of God, Kedushah, Knowledge...). לכן אשכנז מסודר ועובד טוב.
- **`Siddur Sefard`** (=אשכנזי נוסח ספרד, לא ספרדי) → העמידה היא **leaf אחד** בספריא,
  אבל **הטקסט עצמו מכיל כותרות `<b>שם-ברכה</b>`** כפסקאות נפרדות. אפשר ואפשר לפצל!
- אומת ב-API: `curl https://www.sefaria.org/api/v2/index/Siddur_Sefard` ו-`Siddur_Ashkenaz`.

## 3. ⭐ המשימה הגדולה הפתוחה: מפצל-קטעים לעמידת ספרד

**המטרה:** לפצל את ה-leaf המונוליטי של עמידת ספרד לפי כותרות `<b>` לתת-קטעים
וירטואליים עם שמות ברכה (כמו אשכנז), כך שכל הלוגיקה הקיימת (הסתרת קדושה בלחש,
כפתור חזרת ש"ץ, סדר, תוויות) תעבוד גם בספרד. אותו דבר ל-עדות-מזרח וחב"ד אם אותו מבנה.

**גישה:** "לחשוב הפוך" — קודם רשימת הקטעים כמו ספריא קוראת להם, ואז כלל "מה ומתי" לכל אחד.

### מפת קטעים — עמידת ספרד שחרית (מתוך Sefaria, `Siddur Sefard, Weekday Shacharit, Amidah`)

הטקסט = 125 פסקאות. כותרות `<b>` ותגיות `<small>` בסדר:

| כותרת `<b>` | סוג | כלל "מה ומתי" |
|---|---|---|
| אבות | ברכה (לחש+חזרה) | תמיד · זכרנו לחיים — בעשי"ת בלבד |
| גבורות | ברכה | תמיד · מוריד הטל (קיץ) / משיב הרוח ומוריד הגשם (חורף) · מי כמוך — בעשי"ת |
| **קדושה** (נקדישך, קדוש קדוש קדוש, ברוך כבוד, ימלך) | **חזרה בלבד** | רק בחזרת ש"ץ — **מוסתר בלחש**. תוויות קהל/חזן/ש"ץ → הערה דקה |
| קדושת השם | ברכה | תמיד · האל הקדוש / המלך הקדוש (בעשי"ת) |
| בינה | ברכה | תמיד |
| תשובה, סליחה | ברכות | תמיד |
| גאולה | ברכה | תמיד · ענינו — בחזרת ש"ץ בתענית בלבד |
| רפואה | ברכה | תמיד (תחנת חולה ב-collapse) |
| ברכת השנים | ברכה | ברך (חורף) / ברכנו (קיץ) + טל ומטר לפי עונה |
| קיבוץ גליות, דין, ברכת המינים, צדיקים, בנין ירושלים, מלכות בית דוד, קבלת תפלה | ברכות | תמיד · המלך המשפט (בעשי"ת, בברכת דין) |
| עבודה | ברכה | תמיד · **יעלה ויבוא** (ר"ח: רֹאשׁ הַחֹדֶשׁ / פסח / סוכות) |
| הודאה | ברכה | תמיד · על הניסים (חנוכה: בימי מתתיהו / פורים: בימי מרדכי) · וכתוב (בעשי"ת) · **מודים דרבנן** — חזרה בלבד |
| **ברכת כהנים** | **חזרה בלבד** | רק בחזרת ש"ץ — מוסתר בלחש (אבל בתענית ציבור הש"ץ אומר — multi-para block כבר מטופל ב-parser) |
| שלום | ברכה | תמיד · בספר חיים (בעשי"ת) |

### צעדי מימוש מוצעים
1. פונקציה `splitMonolithicAmidah(paragraphs)` ב-`siddurParser.ts` או `siddurAugment.ts`:
   מפצלת לפי `<b>header</b>` → מערך תת-קטעים `{name, paragraphs}`.
2. כל תת-קטע הופך ל-FlatLeaf וירטואלי עם `en`/`he` תואמים לשמות אשכנז (כדי ש-
   `isKedushahLeaf`, `isAmidahLeaf` יזהו).
3. לתייג חזרה-בלבד: קדושה, מודים דרבנן, ברכת כהנים → מוסתר בלחש (כמו אשכנז).
4. תוויות קהל/חזן/ש"ץ → `halachic-note` (מוסתר ב"בלי הערות").
5. אותו מנגנון ל-Mincha/Maariv ספרד ו-עדות-מזרח/חב"ד (לבדוק אם אותו מבנה `<b>`).
6. **לבדוק כל שלב בדפדפן בקריאה רציפה** — לא חיפוש מילים.

- נקדישך מופיע בעמידת לחש (קדושה לא מוסתרת)

- אין כפתור חזרת ש"ץ
- הקדיש שאחרי הלל מתחיל מ"תתקבל" — חסרה ההתחלה (קדיש שבור)

  ה-Song of the Day ב-Sefaria, כנראה צריך פילטר ליום + ניקוי**
- אין קישור לקריאת התורה (בספרד RC)
- שברי קדיש מפוזרים במקומות שונים
